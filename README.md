# SQ Motors UI (SEO + LLM Discoverability Ready)

This project is a React + TypeScript + Vite frontend for SQ Motors.

It now includes a build-time SEO and LLM discoverability system that works with the current inventory setup (direct Facebook inventory links) and stays compatible with a future structured inventory backend.

## Core Architecture

- App framework: `React + TypeScript + Vite`
- Inventory source today: SQ Motors Facebook page / Marketplace listings
- Canonical domain: `https://www.sqmotorsllc.com`
- Single SEO source of truth: [`src/config/seo.config.json`](src/config/seo.config.json)

## Canonical Domain + SEO Config

Canonical settings, local pages, metadata, business facts, and `sameAs` links are centrally managed in:

- [`src/config/seo.config.json`](src/config/seo.config.json)
- Typed access/helpers: [`src/config/seo.ts`](src/config/seo.ts)

Update this config to change:

- canonical host
- route titles/descriptions
- service area pages
- business NAP/hours
- `sameAs` profiles (Facebook, Google Business Profile, Yelp, Cars.com, CarGurus)

Only include verified public URLs in `sameAs`.

## Generated SEO/LLM Assets

Generated by build scripts into `public/`:

- `public/robots.txt`
- `public/sitemap.xml`
- `public/llms.txt`
- `public/.well-known/llms.txt`
- `public/llm/index.md`
- `public/llm/*.md` (core page mirrors)
- `public/llm/locations/*.md` (service-area mirrors)

Generated build freshness file:

- [`src/generated/seoBuild.ts`](src/generated/seoBuild.ts)

## Commands

- Install dependencies:

```bash
npm install
```

- Generate SEO/LLM assets manually:

```bash
npm run seo:generate
```

- Validate SEO setup:

```bash
npm run seo:validate
```

- Full production build (runs generation + validation first):

```bash
npm run build
```

## Inventory Constraints (Current)

Current implementation intentionally does **not** create vehicle detail pages or vehicle-level schema.

- No backend assumed
- No scraping rendered Facebook iframe HTML
- Inventory remains listing-feed based until structured per-vehicle data exists

## Future Structured Inventory Upgrade Path

When a real inventory backend is available:

1. Add typed vehicle records (VIN/trim/year/price/status/images/specs)
2. Generate canonical vehicle detail routes
3. Add vehicle-level JSON-LD (`Vehicle`, `Offer`) from real data
4. Include vehicle URLs in sitemap + llm mirrors
5. Keep current config-driven SEO framework and extend it with vehicle route config

## Dev Notes
